# run-007 — Bonnie v4-clean: Correcção de Data Leakage (2026-05-24)

## Motivação

Diagnóstico de data leakage no walk-forward da Bonnie v4:

1. **CRÍTICO — Label leakage cross-boundary**: Observações de treino nos últimos
   ~20 dias úteis antes de `VAL_START` (2024-12-04 → 2024-12-31) tinham labels
   calculados com preços de Janeiro 2025 (período de validação).
2. **ALTO — Gap = 0 dias**: `TRAIN_END = VAL_START = 2025-01-01` sem buffer.
3. **MÉDIO — Thresholds optimizados no val set**: Per-regime thresholds seleccionados
   no mesmo set usado para reportar métricas → métricas ligeiramente infladas.

## Correcções Aplicadas (`scripts/retrain_bonnie.py`)

| Problema | Correcção |
|----------|-----------|
| Label leakage | `TRAIN_END = datetime(2024, 11, 28)` — 25bd antes de VAL_START |
| Gap zero | Gap efectivo 25bd > LABEL_HORIZON_DAYS=20 — sem leakage |
| Threshold leakage | val_df dividido em val_tune (70%) + test_df held-out (30%) |
| Bug CLI `--val-start` | Removida sincronização automática `TRAIN_END = VAL_START` |

Ficheiros também modificados: `scripts/backtest.py` (suporte `--bonnie-v4clean`).
