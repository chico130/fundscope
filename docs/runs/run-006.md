# Run 006 — Bonnie v4 (Labels Calibradas 4.25×ATR)

**Data:** 2026-05-24  
**Versão:** v3.1  
**Período de validação (OOS):** 2024-01-01 → 2026-05-01  
**Capital inicial:** EUR 5,000  

---

## Contexto

Bonnie v3 foi rejeitada em run-005 por **label mismatch**: treinada com `TP=1.5×ATR` mas a estratégia usa `TP=4.25×ATR`. O modelo tornava-se pass-through (filtrava <0.3% dos sinais).

Bonnie v4 corrige o problema usando labels calibradas para os parâmetros actuais da estratégia.

---

## Configuração do Retrain

```
scripts/retrain_bonnie.py --since 2017-01-01 --until 2026-05-01 \
    --model-version v4 --tp-mult 4.25 --sl-mult 1.75
Treino: 2017-01-01 → 2025-01-01
Val:    2025-01-01 → 2026-05-01
Labels: 4.25×ATR TP / 1.75×ATR SL / 20 dias
Model:  GradientBoostingClassifier (max_depth=3, n_estimators=200)
```

### Análise do Label Balance

Com `TP=4.25×ATR` e horizonte de 20 dias, apenas **15.8%** dos sinais atingem o TP. Com label balance tão baixo:
- O modelo aprende a prever "não" quase sempre → accuracy=84% @ 0.50 (trivial)
- F1=0 @ 0.50 porque Recall=0% (nunca prevê positivo)
- Threshold optimizer encontra 0.30 como melhor F1, mas passes muito baixos no val set

### Paradoxo aparente

As métricas de validação do corpus indicavam 2.6% pass rate, mas no backtest real v4 filtra **32.6%** dos sinais (pass rate = 67.4%). Explicação:

A distribuição dos sinais do corpus e dos sinais reais do backtest é diferente:
- Corpus: `RSI_CEILING=35.0` vs estratégia real: RSI ≤ 34 (Learner)
- O motor de backtest aplica filtros adicionais (posições abertas, sector caps, etc.)
- Sinais reais têm distribuição de features ligeiramente diferente → o modelo assign ≥ 0.30 para ~67% dos sinais reais vs 2.6% do corpus val

**Conclusão:** As métricas de corpus val são pessimistas. O teste real (OOS backtest) é o árbitro correcto.

## Lições

1. **Label balance é o indicador chave.** v3 (45% positivos) = pass-through. v4 (15.8% positivos) = modelo aprende "não" → métricas de corpus val pessimistas mas comportamento real excelente.

2. **Corpus val e sinais reais têm distribuições diferentes.** RSI ceiling 35 vs 34, filtros de posição, etc. fazem com que o modelo seja mais permissivo em prática do que no val set. As métricas de corpus val (F1=0, passes=2.6%) não predizem o comportamento real — o OOS backtest é o único árbitro válido.

3. **Threshold=0.30 funciona.** Mesmo com F1 baixo no corpus val, em prática filtra 32.6% dos sinais e melhora todas as métricas.

4. **Próximo refinamento (Bonnie v5):** aumentar `LABEL_HORIZON_DAYS` de 20 para ~57 dias (`ceil(4.25/1.5 × 20)`) para dar ao TP mais tempo para ser atingido e aumentar o label balance para ~30-40%. Isso deve melhorar as métricas de corpus val sem alterar os labels de negócio.
